# Git/GitHub Workshop - my readme is better

Aim: turn a capstone folder into a clean repo with useful history and a clear next task list.

## Why This Is Here

Planning agents are useful only if the project around them is organized. This
workshop turns the morning's planning idea into a practical repo workflow:
small commits, readable project state, safe environment handling, and GitHub
tasks that teammates can actually pick up.

## Workshop Flow

| Time        | Activity                                                          |
| ----------- | ----------------------------------------------------------------- |
| 13:30-13:50 | Mental model: working tree, staging area, commits, remote         |
| 13:50-14:25 | Local Git workflow with a small file change                       |
| 14:25-15:15 | GitHub workflow: repo, remote, branch, push, pull request, issues |
| 15:30-16:30 | Capstone repo clinic                                              |
| 16:30-17:15 | Team task: prepare a clean commit and PR or PR-ready branch       |
| 17:15-17:25 | Wrap checklist                                                    |

## Student Task

Use your capstone project folder or create a small practice folder.

Complete:

1. Initialize or open a Git repo.
2. Add or improve `README.md`.
3. Add `.env.example`.
4. Add `.gitignore`.
5. Make one small commit.
6. Create three GitHub issues or write a local issue list.
7. Create a branch for the next task.
8. Push and open a pull request, or stop at a PR-ready branch if auth is not ready.

## Success Criteria

- `git status` is clean after the commit.
- `git log --oneline -3` shows a meaningful commit message.
- No API keys or local `.env` values are committed.
- The README says what the project does, how to run it, and what is next.
- The team has three concrete next tasks.
